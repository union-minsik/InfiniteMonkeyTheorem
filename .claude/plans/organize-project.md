## 요약

## Step 1: CLAUDE.md 생성
**왜?** Claude가 이 프로젝트에서 작업할 때 참고하는 규칙서입니다. `.cursorrules`에 이미 좋은 규칙이 있으니 이를 기반으로 Claude용으로 작성합니다.

**내용:**
- 프로젝트 개요 (게임 설명, 기술 스택)
- 폴더 구조 설명 (어디에 뭐가 있는지)
- 코드 스타일 규칙 (TypeScript strict, import 순서 등)
- 실행/빌드/배포 명령어
- 게임 로직 핵심 흐름 (gameStore → useGameLoop → page.tsx)

## Step 2: README.md 업데이트
**왜?** 현재 README는 너무 간단해서 프로젝트 전체를 이해하기 어렵습니다.

**새 README 구성:**
- 프로젝트 소개 + 게임 설명
- 스크린샷/데모 링크 자리 (placeholder)
- 시작하기 (설치 → 실행 → 빌드 → 배포)
- 기술 스택 상세
- 폴더 구조 트리 + 각 폴더 역할 설명

- 새 게임 만들기 가이드 (이 프로젝트를 템플릿으로 쓰는 방법)

## Step 3: 폴더 구조 정리
**왜?** 현재는 게임 코드와 공통 인프라가 섞여 있어서, 나중에 새 게임을 만들 때 어디를 바꿔야 하는지 알기 어렵습니다.

**변경 사항:**
- `src/components/game/` → 게임 전용 컴포넌트 (이미 분리되어 있음, 유지)
- `src/stores/gameStore.ts` → 이미 잘 분리됨, 유지
- `src/config/missions.ts` → 게임 전용, 유지
- `src/hooks/useGameLoop.ts` → 게임 전용, 유지
- `src/lib/sounds.ts` → 게임 전용, 유지

현재 구조가 이미 꽤 잘 분리되어 있습니다. 큰 파일 이동 없이:
- 문서에서 "이 파일은 게임 전용 / 이 파일은 공통"을 명확히 표시
